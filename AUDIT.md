### 1b. Schema simplification (safe once Postgres-only)

- `idColumn` becomes a constant: `INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- `country` ALTER collapses to the single Postgres `ADD COLUMN IF NOT EXISTS` line.
- **DECISION-B (small):** the `country` backfill `UPDATE`s (`schema.js:45-48`) are a
  migration for pre-existing rows. `country` is already in the `CREATE TABLE`
  (`schema.js:20`) and the seed sets it, so for a fresh Postgres DB these are
  dead. Propose deleting them; keep only if we care about already-deployed rows.

### 1c. `?`→`$n` translation & `RETURNING id` (DECISION-C — recommend KEEP)

All SQL across `services/` and `seed.js` is written with `?` placeholders and
auto-translated by `toPostgresSql` (`index.js:37-40`); inserts get their id via
`sqlWithReturningId` (`index.js:42-46`). Options:
- **Keep (recommended):** small, well-tested, zero churn. The functions are not
  SQLite-specific — they're the Postgres adapter.
- Migrate to native `$1,$2…` + explicit `RETURNING id` in every query: touches
  ~30 statements, error-prone (positional), no real benefit. Not recommended.

`hasPostgresConfig` can go because `postgresConfig()` already throws a clear
"Missing Postgres configuration" error when env is absent.

### 1d. ⚠️ DECISION-A — Test database strategy (blocks Phase 3)

Removing SQLite breaks how `index.test.js` (integration) gets a DB. Options:

| Option | Pros | Cons |
|--------|------|------|
| **A1. CI `services: postgres` + local docker Postgres** (tests read `POSTGRES_*`) | Most faithful to prod; no emulation gaps | CI edit; local devs need Postgres up; tests not fully hermetic |
| **A2. Testcontainers** (`@testcontainers/postgresql`) | Real Postgres, hermetic per run | New dev dep; needs Docker locally + in CI; slower startup |
| **A3. `pg-mem`** (in-JS Postgres) | Fast, no Docker, hermetic | New dev dep; dialect gaps (IDENTITY, `ADD COLUMN IF NOT EXISTS`) may need shims |
| **A4. Split** — keep unit tests pure; gate integration tests behind Postgres availability | Minimal infra | Integration tests skipped when no PG → weaker signal |

**Recommendation:** A1 (CI Postgres service + a `docker compose` Postgres for
local), optionally A2 if we want hermetic local runs. Needs the group's call
before Phase 3 implementation.
